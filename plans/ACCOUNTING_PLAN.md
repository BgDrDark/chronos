## 3. ФАЗИ ЗА ИМПЛЕМЕНТАЦИЯ

### ФАЗА 1: PDF Генерация за Фактури 🔴

**Цел:** Професионални PDF документи за фактури

**Стъпки:**
1. Backend endpoint `/export/invoice/{id}/pdf`
2. GraphQL мутация `getInvoicePdfUrl`
3. Frontend бутон "PDF" в списъка с фактури

**PDF Структура:**
```
┌────────────────────────────────────┐
│  ФИРМА ЕООД                       │
│  гр. София, ул. Пример №1        │
│  ЕИК: 123456789                   │
│  ИН по ДДС: BG123456789           │
├────────────────────────────────────┤
│  ФАКТУРА №: INV-2024-0001        │
│  Дата: 20.03.2026                 │
│  📋 ОРИГИНАЛ                      │
├────────────────────────────────────┤
│  ДОСТАВЧИК:              КЛИЕНТ:  │
│  "Пример ООД"           "Клиент АД"│

│  № │ Артикул      │ Кол-во │ Сума │
```

### ФАЗА 2: Автоматични Счетоводни Записи 🔴

**Цел:** При създаване/плащане на фактура → автоматично генериране на записи в сметкоплана

**Стъпки:**
1. Добавяне на default сметки в Company
2. AccountingService за автоматични записи
3. Интеграция в mutations

**Логика:**
```
Входяща фактура (покупка):
├── Дт: 60x (разходи) ← 401 (доставчици)
└── Кт: 401 (доставчици)

Изходяща фактура (продажба):
├── Дт: 411 (клиенти) ← 701 (приходи)
└── Кт: 701 (приходи)
```

---

### ФАЗА 3: Складова Интеграция 🔴

**Цел:** При избор на артикул от склад → показване на партиди с наличност

**Стъпки:**
1. GraphQL заявка `batchesWithStock(ingredientId)`
2. UI промени в Invoice Dialog
3. Намаляване на наличност при запазване

**UI:**
```
┌─────────────────────────────────────────┐
│  Избери партида от склада:             │
│  ○ Партида #001 - 50 бр. - 10.00 лв  │
│  ● Партида #002 - 25 бр. - 10.50 лв   │
│  ○ Партида #003 - 100 бр. - 9.00 лв   │
└─────────────────────────────────────────┘
Количество: [25_____] от наличност: 25
```

---

### ФАЗА 4: Банково Съпоставяне 🟡

**Цел:** Автоматично свързване на плащания с фактури

**Логика:**
1. При качване на банково извлечение
2. Търсене на съответстващи фактури по:
   - Сума (точно съвпадение)
   - IBAN на получател
   - Дата (в рамките на 3 дни)
3. Предлагане на съпоставяне на потребителя

---

### ФАЗА 5: Е-подпис и Легални Изисквания 🟡

**Цел:** Интеграция с e-signature provider

**За България:**
- QR код с линк за валидация
- BARCODE с UIN (уникален идентификатор)
- Печат на НАП
- Timestamp от квалифициран доставчик
