# Правила за Работа с Фактури

**Дата:** 2026-03-27  
**Статус:** Финален

---

## 1. ИЗТРИВАНЕ

### Правило
```
❌ ЗАБРАНЕНО ЗА ВСИЧКИ ФАКТУРИ (НЕЗАВИСИМО ОТ СТАТУСА)
```

### Обяснение
- Изтриване на фактури е **пълно забранено** за всички фактури, независимо от статуса.
- Дори администратори и супер-админи **не могат** да изтриват фактури.

### Какво да правим вместо изтриване:

| Статус | Вместо изтриване |
|--------|------------------|
| draft | Просто не го публикувайте |
| sent | Анулирайте |
| paid | Анулирайте (accountant+) |
| cancelled | Не правете нищо |
| corrected | Не правете нищо |

---

## 2. РЕДАКТИРАНЕ

### Правила
```
┌────────────────────────────────────────────────────────┐
│ draft     → ✅ ПЪЛНО РЕДАКТИРАНЕ                    │
│ sent      → ✅ ПЪЛНО РЕДАКТИРАНЕ                    │
│ paid      → ❌ READONLY                             │
│ cancelled → ❌ READONLY                             │
│ corrected → ❌ READONLY                             │
└────────────────────────────────────────────────────────┘
```

### Обяснение

| Статус | Редактиране | Обяснение |
|--------|-------------|-----------|
| draft | ✅ Да | Нова фактура, мож да се редактира |
| sent | ✅ Да | Изпратена, но не платена - мож да се редактира |
| paid | ❌ Не | Платена фактура = READONLY |
| cancelled | ❌ Не | Анулирана фактура = READONLY |
| corrected | ❌ Не | Коригирана фактура = READONLY |

---

## 3. КОРИГИРАНЕ

### Правила
```
┌────────────────────────────────────────────────────────┐
│ draft/sent → ❌ НЕ (първо платете)                   │
│ paid      → ❌ НЕ (READONLY)                        │
│ cancelled → ❌ НЕ (READONLY)                        │
│ corrected → ❌ НЕ (READONLY)                        │
└────────────────────────────────────────────────────────┘
```

### Обяснение
- Коригиране на фактури е **забранено**.
- Вместо корегиране се издава **Нова корегирана фактура** (кредитно/дебитно известие).
- Коригира се с **нова фактура с коригираните суми**.

### Как да коригираме:

```
Оригинална фактура (1000 лв + 200 лв ДДС)
    │
    ▼
Нова корегирана фактура (-200 лв - 40 лв ДДС)
    │
    ▼
Коригирана фактура (800 лв + 160 лв ДДС)
```

## 4. ПРОМЯНА НА СТАТУС

### Правила
```
┌────────────────────────────────────────────────────────┐
│ draft     → sent, paid, cancelled                      │
│ sent      → paid, cancelled                          │
│ paid      → ❌ НЕ МОЖЕ ДА СИ ПРОМЕНИ СТАТУСА     │
│ overdue   → paid, cancelled, РЕДАКТИРУЕМА         │
│ cancelled → ❌ НЕ                                    │
│ corrected → ❌ НЕ                                    │
└────────────────────────────────────────────────────────┘
```

### Статус Преходи

```
┌─────────┐     ┌─────────┐
│  draft  │────▶│  sent   │
└────┬────┘     └────┬────┘
     │               │
     │               ▼
     │          ┌─────────┐
     ├─────────▶│  paid   │ ❌ READONLY
     │          └─────────┘     │
     │                             │ НЕ МОЖЕ ДА СИ
     │                             │ ПРОМЕНИ СТАТУСА
     │                             │ (дори cancelled)
     ▼                             │
┌────────────┐                     │
│ cancelled  │                     │
│ ❌ READONLY│                     │
└───────────┘                      │
                                  ▼
                             ПЪЛНО READONLY
                             ЗА ВИНАГИ
```

### Обяснение

| От статус | Към статус | Правило |
|----------|-------------|---------|
| draft | sent | ✅ Да |
| draft | paid | ✅ Да |
| draft | cancelled | ✅ Да |
| sent | paid | ✅ Да |
| sent | cancelled | ✅ Да |
| **paid** | **cancelled** | ❌ **НЕ!** |
| **paid** | paid | ❌ **НЕ!** |
| overdue | paid | ✅ Да |
| overdue | cancelled | ✅ Да |
| overdue | (редактиране) | ✅ Да |
| cancelled | (което и да е) | ❌ Не |

## 5. ПРАВА ЗА АНУЛИРАНЕ

### Кои Могат да Анулират

| Роля | Може да анулира платена фактура |
|------|--------------------------------|
| `super_admin` | ✅ Да |
| `head_accountant` | ✅ Да |
| `accountant` | ✅ Да |
| `admin` | ❌ Не |
| `manager` | ❌ Не |
| `employee` | ❌ Не |

### Съобщение при липса на права

```
"Нямате права да анулирате платена фактура. 
За това действие са необходими права на счетоводител или по-високи."
```

## 7. СИНХРОНИЗАЦИЯ

### При Анулиране на Платена Фактура

```
⚠️ ВНИМАНИЕ

Тази фактура има свързани записи:
• Касова книга: 1 запис
• Счетоводство: 2 записа

Тези записи НЕ се анулират автоматично.
Моля, анулирайте ги ръчно.
```

### При Коригиране

```
✅ АВТОМАТИЧНО

При създаване на корекция:
• Генерират се "обратни" счетоводни записи
• Показва се предупреждение с линк към записите
```

## 8. ЗАЩИТА ЗА 10 ГОДИНИ

### Правило
```
❌ Фактури на възраст < 10 години НЕ могат да се изтриват
```

### Обяснение
- Според българското законодателство, счетоводните документи трябва да се съхраняват **10 години**.
- Фактури по-стари от 10 години могат да бъдат изтрити само ако е изтекъл давностният срок.

## 9. ОБОБЩЕНИЕ

### Пълен Преглед

| Статус | Редактиране | Коригиране | Изтриване | Промяна на статус |
|--------|-------------|------------|-----------|-------------------|
| draft | ✅ | ❌ | ❌ | ✅ → sent/paid/cancelled |
| sent | ✅ | ❌ | ❌ | ✅ → paid/cancelled |
| paid | ❌ READONLY | ❌ READONLY | ❌ | ❌ НЕ |
| cancelled | ❌ READONLY | ❌ READONLY | ❌ | ❌ НЕ |
| corrected | ❌ READONLY | ❌ READONLY | ❌ | ❌ НЕ |

### READONLY Фактури

Фактура става **READONLY** когато:
1. Статус стане **paid**
2. Статус стане **cancelled**
3. Статус стане **corrected**

След като една фактура стане READONLY:
- ❌ Не мож да се редактира
- ❌ Не мож да се коригира
- ❌ Не мож да се изтрие
- ❌ Не мож да си промени статуса
