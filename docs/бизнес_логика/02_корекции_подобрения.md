# План: Подобряване на "Корекции"

**Дата:** 2026-03-27  
**Статус:** ✅ Имплементирано (с изключение на тестване)

---

## Цел

Подобряване на функционалността "Корекции" (Invoice Corrections) в системата:
- Добавяне на поддръжка за дебитни известия
- Автоматично номериране
- Диалог за създаване
- Валидация

---

## Решения

| Въпрос | Решение |
|--------|---------|
| Формат на номера | `КР-{ГОДИНА}-{NNNN}` (напр. `КР-2026-0001`) |
| Създаване на нова фактура | Опционално (по избор на потребителя) |
| Роли | `accountant`, `head_accountant`, `super_admin` |
| Валидация | Да - не може да се коригира вече коригирана/анулирана |

---

## Стъпки за Имплементация

### Фаза 1: Backend - основна логика

#### 1.1 Нова мутация: `create_invoice_correction`

**Файл:** `backend/graphql/mutations.py`

```python
@strawberry.mutation
async def create_invoice_correction(
    self,
    original_invoice_id: int,
    type: str,  # "credit" или "debit"
    reason: str,
    correction_date: datetime.date,
    create_new_invoice: bool = False,
    info: strawberry.Info
) -> types.InvoiceCorrection:
```

**Логика:**
1. Проверка за права: `accountant`, `head_accountant`, `super_admin`
2. Валидация: статус != `cancelled` и != `corrected`
3. amount_diff = + или - сумата според типа
4. Автоматично номериране: `КР-2026-0001`
5. Опционално: създаване на нова фактура

#### 1.2 Автоматично номериране

**Логика:**
1. Прочита се последният номер за текущата компания и година
2. Генерира се нов номер
3. Запазва се в InvoiceCorrection

#### 1.3 Валидация

**Не може да се коригира:**
- Фактура със статус `cancelled`
- Фактура със статус `corrected`

**Съобщения:**
```
❌ "Не можете да коригирате тази фактура. Тя е анулирана."
❌ "Не можете да коригирате тази фактура. Тя вече е коригирана."
```

### Фаза 2: Backend - нова фактура (опционално)

#### 2.1 При create_new_invoice=True

След създаване на корекцията, автоматично се създава нова фактура:
- Копира се клиентска информация
- amount_diff се добавя/изважда от оригиналните суми
- Статус: "draft"
- Маркира се оригиналната като "corrected"
