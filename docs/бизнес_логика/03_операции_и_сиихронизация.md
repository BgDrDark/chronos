## Част 2: Синхронизация

### 2.1 Анулиране на Платена Фактура

**Текущо поведение:**
```
Фактура (paid) ──▶ Касова книга ──▶ Счетоводни записи
    │
    ▼
Потребител анулира фактурата...
    ├──▶ Статус: "cancelled" ✅
    ├──▶ Касова книга: НЕ СЕ ПРОМЕНЯ ❌
    └──▶ Счетоводни записи: НЕ СЕ ПРОМЕНЯТ ❌
```

**Решение:**
1. ❌ НЕ анулираме свързаните записи автоматично
2. ⚠️ ПОКАЗВАМЕ предупреждение: "Тази фактура има свързани записи в Касовата книга и Счетоводството. Моля, анулирайте ги ръчно."

### 2.2 Корекции - Счетоводни Записи

**Текущо поведение:**
```
Фактура ──▶ Счетоводни записи (обикновени)
    │
    ▼
Потребител създава корекция...
    ├──▶ Корекцията се създава ✅
    └──▶ Счетоводни записи: СЕ ГЕНЕРИРАТ ✅
```

**✅ ИМПЛЕМЕНТИРАНО:**

1. **Модел:** Добавени полета `correction_id` и `is_reversal` в `AccountingEntry`
2. **Сервиз:** Нов метод `create_correction_entries()` в `AccountingService`
3. **Мутация:** При `create_invoice_correction` се извиква сервизът автоматично

**Пример:**
```
Оригинална фактура: 1000 лв + 200 лв ДДС = 1200 лв

Корекция (кредитно): -1000 лв - 200 лв ДДС

Обратни записи:
  - REV-КР-2026-0001: Дт: 701 / Кт: 411  (1000 лв)
  - REV-КР-2026-0001-VAT: Дт: 453 / Кт: 411  (200 лв)
```
